1 O projektu

Web sajt organizacije International Rotary Fellowship of Healthcare Professionals (IRFHP) — svjetske mreže liječnika, medicinskih sestara i volontera unutar Rotaryja koji provode humanitarne zdravstvene projekte (neonatalna skrb, prevencija raka vrata maternice, kirurške misije, Ukrajina, sigurnost pacijenata i dr.).

Sajt je statični web (brz, siguran, jeftin) izgrađen alatom Astro, s CMS-om za vijesti (Keystatic) tako da administratori mogu sami dodavati novosti bez programiranja. Cilj: moderan, kredibilan sajt uz jednostavno održavanje.

Vidljivost preview verzija (Vercel Authentication)

Vercel po defaultu štiti deploye postavkom Deployment Protection → Vercel Authentication. Kod ovog projekta uključena je za:

  • sve preview deploye (svaka grana osim produkcijske — npr. cms-news), i
  • generirani produkcijski *.vercel.app URL.

Tko otvori takav link, a nije ulogiran u Vercel tim Projekti_B, dobije Vercel login stranicu. Zato vanjski ljudi (npr. Rotary odbor) ne mogu vidjeti preview dok im se ne podijeli na jedan od načina niže. Nije povezano s DNS-om — to je sigurnosna postavka Vercela, ne stvar domene; vrijedi i prije i poslije DNS cutovera.

6 Struktura projekta

rotary-website/ ├─ public/ ← statični resursi (serviraju se s korijena /) │ ├─ images/… ← sve slike (mirror WP uploads: /images/GGGG/MM/…) │ └─ style.css ← cijeli dizajn-sistem (CSS) ├─ src/ │ ├─ layouts/Base.astro ← zajednički header, nav, footer, <head> │ ├─ pages/ │ │ ├─ index.astro ← naslovnica (hero slideshow, impact, latest news) │ │ ├─ about.astro, kmc.astro, … (23 podstranice) │ │ └─ news/ │ │ ├─ index.astro ← popis vijesti (/news) │ │ └─ [slug].astro ← pojedina vijest (/news/<slug>) │ ├─ content/news/*.mdoc ← 17 vijesti (frontmatter + tekst) │ ├─ content.config.ts ← definicija news kolekcije (Astro) │ └─ keystatic/Brand.tsx ← logo u adminu (link natrag na web) ├─ keystatic.config.ts ← definicija CMS-a (polja vijesti, storage) ├─ astro.config.mjs ← Astro + integracije + Vercel adapter ├─ vercel.json ← framework: astro ├─ scripts/ ← jednokratne migracijske skripte │ ├─ migrate.mjs ← stari HTML → Astro stranice │ └─ news-import.mjs ← uvoz vijesti iz WordPress API-ja └─ docs/project-overview.html ← ovaj dokument

Kako radi routing

  • Svaka datoteka u src/pages/ postaje ruta: about.astro → /about, index.astro → /.
  • news/[slug].astro je dinamička ruta — Astro pri buildu generira po jednu stranicu za svaku vijest iz kolekcije.
  • Zajednički izgled (header/footer) dolazi iz Base.astro — mijenja se na jednom mjestu.
  • public/ se servira s korijena: public/style.css → /style.css, public/images/x.jpg → /images/x.jpg.

7 CMS — vijesti (Keystatic)

Vijesti se uređuju kroz Keystatic admin na ruti /keystatic. Svaka vijest je jedna datoteka src/content/news/<slug>.mdoc — versionirana u gitu (nema baze podataka).

Polja vijesti

PoljeTipOpis
Naslovtekst (slug)Naslov + iz njega URL vijesti
DatumdatumZa sortiranje i prikaz
SažetaktekstKratki opis na popisu
Naslovna slikaslikaThumbnail + cover (opcionalno)
TekstmarkdocBogati tekst (bold, linkovi, liste, slike)

Kako admin doda vijest

  1. Otvori /keystatic → Vijesti (News) → Create.
  2. Ispuni polja, klikni Save.
  3. Izmjena se sprema kao commit u repo → Vercel automatski rebuilda → za ~1 min vijest je gore.
Trenutno stanje CMS-a: storage: local — admin radi samo lokalno (npm run dev). Da admini uređuju uživo preko GitHub logina, potrebna je Faza 2 (vidi Što slijedi).

9 Sadržaj

  • 23 podstranice migrirane iz starog sajta u Astro (About, Service projects, KMC, Ukraine, Membership, Contact…).
  • 17 vijesti uvezeno iz WordPressa preko REST API-ja (/wp-json/wp/v2/posts?categories=2&_embed): puni tekst (HTML→Markdown), datumi, sažeci, istaknute slike.
  • Sve slike lokalizirane u public/images — bez ovisnosti o starom sajtu.
  • _template.html je predložak za podstranice i namjerno se ne objavljuje (u .gitignore).

12 Što slijedi

Faza 2 — CMS uživo (GitHub login)

  1. Prebaciti keystatic.config.ts na storage: { kind: 'github', repo: … }.
  2. Kroz /keystatic čarobnjak napraviti GitHub App za autentikaciju.
  3. Upisati env varijable u Vercel: KEYSTATIC_GITHUB_CLIENT_ID, KEYSTATIC_GITHUB_CLIENT_SECRET, KEYSTATIC_SECRET, PUBLIC_KEYSTATIC_GITHUB_APP_SLUG.
  4. Dodati prave administratore kao suradnike (write) na repo.

Faza 3 — uputa adminima

Kratki vodič „kako dodati/urediti vijest" (najbolje i na engleskom, jer je organizacija međunarodna).

Završno

  • Pregled izgleda i sadržaja → merge cms-news u main.
  • Kad se odluči: DNS cutover — kod DNS providera apex → 76.76.21.21, www → cname.vercel-dns.com.
