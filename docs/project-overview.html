2. Arhitektura

Pages CMS → GitHub repo → GitHub Actions → GitHub Pages
  • Pages CMS uređuje Markdoc datoteke i slike izravno u GitHub repozitoriju.
  • Promjena na main pokreće .github/workflows/deploy-pages.yml.
  • Astro generira statički dist/; nema baze, servera ni dugotrajnih tajni.
  • PayPal, Jotform i MemberMojo ostaju vanjske usluge.

Rad ide kroz branch i Pull Request. Merge na main objavljuje website kada je repozitorijska varijabla ENABLE_GITHUB_PAGES=true.

5. CMS tijek

  1. Urednik otvara app.pagescms.org.
  2. Odabire repo i kolekciju Vijesti.
  3. Uređuje naslov, datum, sažetak, naslovnu sliku i sadržaj.
  4. Save izrađuje GitHub commit; deploy se pokreće automatski.
  5. Urednik nakon objave provjerava vijest na javnoj domeni.
Pages CMS GitHub App treba ograničiti samo na repo website. Karla i Drago ostaju vlasnici organizacije; drugi urednici mogu se pozvati kao CMS suradnici.

7. Preostali vanjski koraci

  1. GitHub organizaciju prijaviti za besplatni Team for Nonprofits.
  2. Instalirati Pages CMS GitHub App samo na website repo.
  3. Karla mora dodati i spremiti probnu vijest.
  4. U GitHub Pages postavkama dodati custom domenu i uključiti workflow varijablu.
  5. Prema deployment vodiču promijeniti DNS i uključiti HTTPS.
Ne gasiti Vercel ni Bluehost prije uspješnog Pages deploya, CMS testa i najmanje 48 sati stabilnog rada domene.

8. Ownership

  • GitHub organizacija i repo pripadaju IRFHP-u.
  • Karla i Drago trebaju biti Owneri organizacije s uključenim 2FA.
  • Developer dobiva samo potreban repo pristup.
  • DNS i domena ostaju na organizacijskom računu, uz dokumentiran recovery.
  • Nema produkcijskih tajni u repozitoriju.
