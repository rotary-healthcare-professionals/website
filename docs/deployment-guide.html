GitHub Pages deploy i DNS

Operativna uputa za aktivaciju, provjeru, DNS cutover i rollback IRFHP websitea.

Preduvjeti

  • GitHub Team mora biti aktivan za privatni repozitorij; preporuka je besplatni GitHub Team for Nonprofits.
  • Workflow .github/workflows/deploy-pages.yml mora biti na main grani.
  • npm run build mora završiti bez greške.
  • Postojeći Vercel deployment ostaje aktivan do završne provjere.

Aktivacija GitHub Pagesa

  1. Repo → Settings → Pages → Source: GitHub Actions.
  2. U Pages postavkama dodaj custom domenu rotaryhealthprofessionals.org.
  3. Repo → Settings → Secrets and variables → Actions → Variables.
  4. Postavi ENABLE_GITHUB_PAGES=true.
  5. Actions → Deploy to GitHub Pages → Run workflow.
  6. Provjeri da su build i deploy zeleni prije DNS izmjene.

Provjera nakon cutovera

  • probna CMS izmjena i novi automatski deploy

Rollback

Ako kritična funkcionalnost ne radi, vrati prethodne DNS zapise, uključujući apex 50.87.253.47, i pričekaj propagaciju. Ne uklanjaj Vercel projekt ni stari hosting najmanje 48 sati nakon uspješnog cutovera.
